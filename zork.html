Zork I on AWS EC2 Live

Stack: AWS EC2  ·  AWS S3  ·  IAM Roles  ·  CloudWatch  ·  Python  ·  Flask  ·  WebSocket  ·  dfrotz

The terminal below is running live on an AWS EC2 instance. When you type a command and hit Enter, it gets sent over a WebSocket connection to a Python server on that instance, which passes it to a Z-machine interpreter running Zork I. The game file lives in an S3 bucket and is pulled down at server startup using the EC2's IAM role.

The point wasn't just to play Zork. It was to have something real running on AWS that I could point to and explain. Certificates tell you someone sat through a course. This is what actually sticking around long enough to make something work looks like.

EC2 Amazon Linux 2023 instance running the Flask WebSocket server and dfrotz interpreter.
S3 Game file (zork1.z5) stored in a private S3 bucket, pulled to the instance at startup.
IAM EC2 instance role grants read access to S3. No credentials in code.
CloudWatch CPU and memory monitoring with SNS alerts if the instance gets hammered.

Type commands and press Enter. Each session starts a fresh game. Classic text adventure from Infocom, 1980. If you've never played: i'd suggest looking in the mailbox.
