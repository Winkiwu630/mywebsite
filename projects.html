Self-Hosted Homelab Active

Stack: Ubuntu Server  ·  Docker  ·  Podman  ·  Cloudflare Tunnel  ·  Cloudflare Zero Trust  ·  systemd  ·  Nginx  ·  Tailscale

A production-grade home server running 10+ self-hosted services including a media stack, request management, and remote access infrastructure. Designed and maintained entirely from scratch. No pre-built NAS OS or turnkey solutions.

  • Zero-exposure public access via Cloudflare Tunnel (no open ports)
  • Cloudflare Zero Trust policies restrict access to trusted users and devices
  • Mix of Docker Compose and rootless Podman + systemd services
  • Services include: Jellyfin, Audiobookshelf, Jellyseerr, qBittorrent, and more
  • Automated media pipeline: request → download → rename → library scan → notify
  • 3-2-1 backup strategy for irreplaceable data: 3 copies, 2 locations, 1 offsite (Backblaze B2 via rclone)
  • Ongoing, regularly extended with new tooling and hardening

Guppi | AI Home Assistant Active

Stack: Python  ·  Claude API (Anthropic)  ·  Telegram Bot API  ·  Podman  ·  systemd  ·  REST APIs  ·  webhooks

Built with AI-assisted code review.

A personal AI assistant running as a containerized service on my homelab. Guppi monitors my self-hosted media stack and proactively notifies me through Telegram when services need attention without me having to ask. It handles media requests, triggers library scans after file operations, and integrates with my Audiobookshelf and Jellyfin instances via their REST APIs.

  • Integrates Claude API for natural-language interaction and reasoning
  • Webhook-driven event handling that responds to system events in real time
  • Runs as a Podman container managed by systemd for reliable, rootless operation
  • Handles cross-service orchestration (rename → scan → notify pipeline)
  • Proactive monitoring: detects and surfaces issues without polling from the user
